# 0.5.0 agent-folder reorganisation plan

**Status:** ✅ executed (see commits 3cf2976, b7c37de, bc0d033, c8db709, 0c5ce3a, 697587f)
**Date:** 2026-06-08
**Operator-locked identity:**

> **JROS = Hermes-in-`agent/` + ROS-in-`nodes/` + a shared `transport/`
> that lets them talk.**

The folder layout must read that way at a glance.  Currently it
doesn't — agentic code is scattered across `core/tools/`,
`core/skills/`, `core/prompts/`, plus `jaeger_os/skills/`,
`jaeger_os/personas/`, and `jaeger_os/prompts/`.  This commit doc
locks the destinations + execution order so the moves are
reversible and reviewable.

## The conceptual map (post-reorg)

```
jaeger_os/
├── agent/              ← THE CONSCIOUS NODE ("the Hermes")
├── nodes/              ← THE UNCONSCIOUS NODES ("the ROS")
├── transport/          ← THE BRIDGE
├── skill_tree/         ← META-SYSTEM (XP across both)
├── personality/        ← character config
├── timeline/           ← performance scheduler
├── core/               ← INFRASTRUCTURE shared by agent + nodes
├── plugins/            ← 3rd-party engine wrappers
├── interfaces/         ← operator surfaces
└── topics.py           ← bus SSOT
```

## Destination map (every move)

| From | To | Notes |
|---|---|---|
| `jaeger_os/personas/`   | `jaeger_os/agent/personas/`   | YAMLs only; zero Python imports |
| `jaeger_os/prompts/`    | `jaeger_os/agent/prompt_assets/` | single markdown asset |
| `jaeger_os/skills/`     | `jaeger_os/agent/skills/`     | v3 bundles (operator content) |
| `jaeger_os/core/prompts/` | `jaeger_os/agent/prompts/`  | assemble.py + rules.py |
| `jaeger_os/core/skills/`  | `jaeger_os/agent/skill_registry/` | v3 loader / manifest parser |
| `jaeger_os/core/tools/`   | `jaeger_os/agent/tools/`    | 30 modules — the big one |
| `jaeger_os/core/runners/` | `jaeger_os/agent/runners/`  | ThinkingRunner (agent-only) |

## Execution order (bisectable, test-suite-guarded)

Each phase is its own commit so any regression bisects cleanly.

1. **Phase A — content-only moves** (zero Python import touch)
   - `personas/` → `agent/personas/`
   - `prompts/`  → `agent/prompt_assets/`

2. **Phase B — skills v3 bundles**
   - `skills/` → `agent/skills/`
   - Update internal absolute imports
     (`from jaeger_os.skills.X` → `from jaeger_os.agent.skills.X`)

3. **Phase C — core agent extractions** (highest import touch)
   - `core/prompts/`  → `agent/prompts/`
   - `core/skills/`   → `agent/skill_registry/`
   - `core/tools/`    → `agent/tools/` (30 files — broken into
                                        sub-commits if the diff is
                                        unwieldy)
   - `core/runners/`  → `agent/runners/`

4. **Phase D — docs**
   - `STRUCTURE.md` rewritten around Hermes/ROS/Transport identity
   - `agent/README.md` expanded
   - `README.md` (root) gets the one-line pitch
   - `CHANGELOG.md` entry
   - `agent_contract.md` regenerated

## Verification gates

- `pytest dev_tests -q` passes after every phase (no behaviour change)
- `./launch --tts-boot-test` PASS
- `./launch --node-test` PASS
- `./jaeger skills` still renders (CLI consumes the same data)
- No backwards-compat shims — clean rename, all consumers updated

## Out of scope

- Renaming files within unchanged directories (cosmetic — net negative)
- Collapsing `agent/schemas/` to one file (rejected — distinct concerns)
- C++ extensions in `nodes/motor/` or `nodes/vision/` (0.6 work)
- Making the agent a real Node subclass (separate behaviour refactor)

## Reversal

Per phase, `git revert <hash>` cleanly undoes that step.  Branch
stays at `origin/0.5.0` throughout; no protected refs touched.
