### Finding 1 — TUI voice bus migration (release blocker)

`jaeger_os/interfaces/tui/voice_session.py` currently calls
`_get_tts()` and uses `_tts.warm()`, `_tts.stop()`,
`_tts.play_async()`, `_tts.wait_until_done()`, `_tts.speak()`
directly.  Migrate them to bus calls so the TUI voice path
actually exercises `/act/speech`, `/act/speech_stop`,
`/sense/spoken` (which the standalone `voice_loop.py` already
does at commit `4cb9597`).

**Pattern to follow** (already proven in `voice_loop.py`):

```python
# At controller construction:
from jaeger_os.nodes import runtime
from jaeger_os.transport import topics
self._bus = runtime.get_bus()

# Replace _tts.warm():
runtime.ensure_tts_node(warm=True)

# Replace sync _tts.speak(text):
import uuid
cid = uuid.uuid4().hex
ack = self._bus.request(
    topics.SpeechCommand(
        text=text, node_id="tui-voice",
        correlation_id=cid,
    ),
    ack_topic=topics.SENSE_SPOKEN,
    timeout_s=180.0,
)
# ack is None on timeout; ack.ok=False on synth failure or barge-in

# Replace play_async + wait_until_done + tts.stop barge-in:
# Install the STT speech-detected callback to publish SpeechStop
# instead of calling _tts.stop() directly.
def _on_user_speaks():
    if not interrupted["flag"]:
        interrupted["flag"] = True
        self._bus.publish(topics.SpeechStop(
            reason="user interrupted",
            node_id="tui-voice",
        ))

self._stt.set_on_speech_detected(_on_user_speaks)
try:
    ack = self._bus.request(...)  # same as sync path above
finally:
    self._stt.set_on_speech_detected(None)
```

Keep the existing return shape (`bool` = whether user barged
in).  Decide barge-in from the `interrupted` flag OR from
`ack.reason == "interrupted"`.

Drop the `from jaeger_os.core.tools.speak import _get_tts`
import + the `self._tts = _get_tts()` assignment.  The TUI no
longer needs a direct synth reference.

Sanity check: `./launch --tts-test` must still pass; the integration
gate already exercises the same bus path.

### Finding 2 — voice_loop fresh-install defaults

`jaeger_os/plugins/voice_loop.py` at line 164 reads:

```python
require_wake_word = not args.no_wake_word
barge_in_requested = not args.no_barge_in
```

This is 0.2.6 CLI semantics — the flag default IS the behavior,
config is ignored.  Fix so config is the source of truth and the
CLI flag overrides only when given.

Pattern (the implementing agent uses this elsewhere):

```python
# argparse: change default from False to None so we can distinguish
# "not passed" from "passed False"
p.add_argument("--no-wake-word", action="store_const",
               const=True, default=None, help="...")
p.add_argument("--wake-word", action="store_const",
               const=True, default=None, help="opt back in")

# Resolution (config wins unless CLI explicitly set):
if args.no_wake_word is not None:
    require_wake_word = False
elif args.wake_word is not None:
    require_wake_word = True
else:
    require_wake_word = bool(config.voice.wake_word)

# Symmetric for barge_in.
```

Result: a fresh install where `config.voice.wake_word = False`
(the 0.4.x default after commit `517d751`) actually gets
wake-word OFF in the standalone voice_loop, matching the TUI.

### Finding 3 — InProcBus publish contract

`jaeger_os/transport/inproc_bus.py:65` uses blocking `put()`.  The
abstract Bus says publish "never blocks beyond write time".  Fix
the implementation, not the contract — the contract is what nodes
rely on.

```python
def publish(self, msg: topics.TopicMessage) -> None:
    if self._closed:
        return
    try:
        self._q.put_nowait(msg)
    except queue.Full:
        # Overflow policy: log + drop the OLDEST message + retry.
        # Document this in the docstring as the chosen overflow
        # strategy.  Alternatively: log + drop the new message.
        # Pick one consistently.
        import sys
        try:
            self._q.get_nowait()
        except queue.Empty:
            pass
        try:
            self._q.put_nowait(msg)
        except queue.Full:
            print(
                f"[inproc-bus] queue full ({self._q.maxsize}); "
                f"dropped message on {msg.topic}",
                file=sys.stderr, flush=True,
            )
```

Update the publish() docstring to describe the overflow policy.
Update the test fixture `dev/tests/jaeger_os/transport/test_inproc_bus.py`
with a new test that forces queue overflow + verifies the policy.

### Finding 5 — SpeechStop correlation tracking (NICE-TO-HAVE)

Two options:

**A.** Document SpeechStop as "stop current speech globally" in
both the schema docstring (`jaeger_os/topics.py`) and the TTSNode
(`jaeger_os/nodes/tts/node.py`).  Remove `correlation_id` from
the SpeechStop schema since it's misleading.

**B.** Track the active correlation_id in TTSNode (set when
`_handle` enters, cleared when SokenAck publishes).  If
SpeechStop has a `correlation_id` AND it doesn't match the active
one, log + ignore.

Pick A unless you have a concrete second-speaker scenario in mind.
A is honest about the current semantics; B is correct-but-currently-
overkill.
