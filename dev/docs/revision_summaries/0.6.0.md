# JROS 0.6.0 — install / update / lifecycle (the product shell)

**Status:** active branch `0.6.0` (not yet tagged) — this summarises what the
branch has delivered so far
**Tag:** `0.6.0` *(pending)*
**Tagline:** JROS *feels like a real app* — install it, run it, keep it current

0.5.0 made the agent alive. 0.6.0 makes the **product around it** feel
finished: a clean one-command install, an in-place update with rollback, a
boot/login service, a clickable launcher, and a reinstall/uninstall that keep
the operator's agents intact. The agentic pipeline also got a round of polish
that landed alongside the theme work.

## What we set out to do

Make JROS behave like software an operator deploys on their own projects and
robots — **not** something you babysit from a dev clone:

1. A clean install, and a *nice* way to update deployed units.
2. The full lifecycle: install → run (manual or unattended) → update → roll
   back → reinstall → uninstall.
3. Stop designing around the contributor's git checkout; design around the
   operator running a clean product install.

## What actually shipped

### Packaging — JROS is a proper package again
- Editable install (`uv pip install -e .`), one real `jaeger` console entry
  point, version single-sourced from `jaeger_os.__version__` (pyproject reads
  it dynamically). The key realisation that unlocked the rest: **the agent
  self-modifies into instance state (`.jaeger_os/`), never the package** — so a
  clean, read-only product install is correct; editable-from-clone is only for
  contributors hacking the framework.

### Update — the headline
- **`jaeger update`** on a clean (no-`.git`) install **downloads the target
  release tarball and swaps the product in place**, preserving `.venv/` +
  `.jaeger_os/`; `--ref TAG` pins, `--rollback` reverts (prev kept), deps
  reinstall only when they change. Dev clones keep the git-pull path.
- **`version_check`** — numeric version compare + `latest_version()` (GitHub
  tags), shared by `update`, `doctor` (current-vs-latest readout), and the
  in-app surfaces.
- **`jaeger reinstall`** — clean in-place reinstall keeping agents (clean
  install → force re-fetch + dep resync; dev clone → repair the editable env).
- **`jaeger uninstall`** — removes the framework, keeps agents unless
  `--purge`, refuses on a dev clone.

### Run unattended + clickable
- **`jaeger autostart enable|disable|status`** — macOS LaunchAgent / Linux
  `systemd --user` (+ linger) so a unit comes up after reboot/power-loss.
- **`jaeger launcher install|remove`** — a thin, locally-created (unsigned, no
  Gatekeeper prompt) `Jaeger.app` in `/Applications`.
- **In-app "update available"** — a tray "Check for Updates…" item + a Jaeger
  Studio banner, both via `version_check.update_status`.

### Operator vocabulary
- **instance → agent** (surface only): `jaeger agent <create|list|use|inspect|
  delete|clear>` unifies the old `setup`/`instance`/`instances`; `--agent`
  flag; old names kept as aliases. Internal `instances/` code unchanged.

### Lighter, cleaner distribution
- Untracked 93 MB of derived Swift `.build/` that had been dragged into every
  clone + install; `.gitattributes export-ignore dev/` trims the release
  tarball. README brought current (canonical `jaeger`, `jaeger update` upgrade
  path, agent vocabulary, version badge).

### Agentic (off-theme, operator-prioritised)
- **Autonomy modes** (`ask` / `scoped` / `auto`) — agree the plan up front,
  then run without per-action prompts; reach out only when blocked.
- **Person index** — profiles of people the agent knows, grown like skills.
- **Admin trust** — per-channel admin gating + in-channel approvals shared
  across Telegram/Discord/iMessage.
- **Skill self-improvement loop (on by default)** — per-use notes → threshold
  trigger → measured Deep Think rewrite (smoke + benchmark, keep-if-better) →
  revision log.
- **Model defaults from a clean 7-model benchmark** — `e4b` awake/voice,
  `26B-A4B QAT` deep-think, `12B` voice backup.

## What we tried and abandoned

1. **Flatten the package / drop the `jaeger_os` namespace** — considered, then
   rejected: an editable package keeps imports clean and updates simple without
   moving a single file.
2. **`pip install jaeger-os` from PyPI as the distribution** — rejected for the
   robot/appliance profile (native deps + a service + model files fight pip;
   the wheel stays *possible* but isn't the operator path). A curl install +
   download/apply update fits units far better.
3. **git-pull as the only update path** — works for clones, not for the clean
   curl install (no `.git`). Replaced by the no-git download/apply updater.

## Lessons learned

- **Design for production, not the dev clone.** The pivotal correction this
  cycle — the operator deploying on a robot, not the contributor hacking the
  framework, is the customer for install/update.
- **Know what self-modifies.** Because the agent writes to instance state and
  never the package, a read-only clean install is safe — that single fact
  resolved the whole flatten-vs-package-vs-wheel debate.
- **Surface vocabulary ≠ internal vocabulary.** Renaming "instance" → "agent"
  for operators without churning the internal `InstanceLayout` code (which
  would collide with the "agent loop" meaning) was the right, cheap scope.
- **Walk the artifact even when you can't walk the GUI.** The launcher `.app`
  and update flow were validated end-to-end (real GitHub tarball, `plutil`-lint
  the plist, offscreen banner) rather than on inspection.

## Patterns absorbed

- **App-style lifecycle** (ollama / Home Assistant): a native installer + an
  in-place updater + a service, not a library `pip install`.
- **Download-and-swap with a kept previous + rollback** — the standard safe
  in-place update shape.

## What 0.6.0 sets up

A unit you can install, run unattended, and keep current without git or PyPI —
the prerequisite for shipping JROS onto real hardware. Next: the in-app update
*action* (not just the notice), `doctor` Full-Disk-Access detection, and the
0.7 hardware-adapter track (JP01 mic/RGB/speaker over the `MediaFrame` /
`ZmqBus` seam).
