# JROS 0.1.0 — first release

**Released:** 2026-05-25
**Tag:** `0.1.0` (commit `d31b703`)
**Tagline:** local-first agentic agent framework

## What we set out to do

Build a local-first agentic AI framework — a Mac-side LLM with
real tools that could do work, not just chat.  No cloud
dependency for the brain.  No "agent of the week" framework
overhead.  An honest, runnable thing that worked on the
operator's actual machine with the operator's actual model.

The 0.1.0 question was simple: *"Can a local Gemma + a small
tool surface actually act like an agent, or does it fall apart
the moment you give it real work?"*

## What actually shipped

* The first version of the agent loop in `main.py` — single
  thread, fast finalize, tool dispatching via JSON.
* A small tool surface: file ops, memory (key/value), web
  search, calculator, weather, time.
* The first persona system — `identity.yaml` + `soul.md`.
* The TUI as the operator-facing surface (no daemon, no GUI).
* The first benchmark: 51 cases across routing / memory /
  multi-turn / safety.  Reference at
  [`BENCHMARK_0.1.0.md`](../BENCHMARK_0.1.0.md).
* Manual model resolution — operator pointed at a GGUF on
  disk; nothing automatic.
* SQLite-free first; everything was JSON / JSONL files in the
  instance directory.

## What we tried and abandoned

This was the first release; nothing was tried and abandoned at
0.1.0 itself.  But the *pre-history* (Alpha 0.1 through Alpha 20)
churned through several agent-loop shapes — see the bottom of
[CHANGELOG.md](../../CHANGELOG.md) for the alpha-era notes.

## Lessons learned

1. **Local Gemma 4 26B-A4B (Q4) is genuinely an agentic model.**
   Not GPT-4-class, but it does multi-step tool chains reliably.
   This validated the whole "local-first agent" thesis.
2. **The tool surface mattered more than the model.**  Switching
   models within a corpus changed the benchmark score by 5-15
   points; cleaning up tool docstrings changed it by 30+.
3. **JSON files for state were enough for one instance.**  Scaling
   to multi-instance later would force the SQLite migration, but
   for 0.1.0 the simplicity won.
4. **The reference voice assistant
   (`dev_tools/audio_smoke/voice_assistant_persistent.py`) was the
   right shape** — small, single-file, no abstractions.  It would
   be the touchstone for the rest of the year.

## Patterns absorbed

* The TUI's per-turn rhythm came from a prior MockingAgent
  experiment.
* The fast-finalize / decide-then-write split came from reading
  Anthropic's tool-use docs.
* Per-instance state in `~/.jaeger` came from looking at how
  `git` separates the repo from `~/.gitconfig`.

## Where 0.1.0 lives in the codebase today

Most of it has been replaced.  The bits still present:

* The persona file shape (`identity.yaml` + `soul.md`) — kept
  forever, just extended.
* The tool docstring conventions — kept forever.
* The 51-case bench corpus, archived under
  `dev/benchmark/archive/` and shown at the bottom of
  `dev/benchmark/HISTORY.md` for comparison against the current
  59-case 1.1 corpus.

Everything else has been refactored — see
[`0.2.0.md`](0.2.0.md) for the architectural cleanup pass that
produced what JROS looks like today.
