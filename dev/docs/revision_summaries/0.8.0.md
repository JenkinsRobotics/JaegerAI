# JROS 0.8.0 — modular runtime + persona_first pipeline

**Status:** branch `0.8.0`, head `34da639` — RC battery green, release docs
in progress; tag/push on operator's explicit OK per standing rules.
**Tag:** `0.8.0` *(pending — not yet cut)*
**Tagline:** one bus, one Node, one runtime — and the id/ego pipeline as
default

0.7.0 made JROS a native Mac app with a measured two-runner core. 0.8.0
resolves the runtime's remaining duality (two bus implementations, two Node
classes) into one, turns TTS/STT/animation/media into swappable
engine-modules ("the module IS the engine"), and ships **persona_first** —
the id/ego pipeline — as the default agentic path, hardened until it beat
its own worker-path baseline on security.

## What we set out to do

1. Finish the modular-framework north star deferred from 0.7: one runtime,
   modules that own their own engine/config/lifecycle, a unified slot-bound
   surface.
2. Make the persona-voice pipeline the default without the latency tax or
   the security regressions a naive frontend/filter approach would carry.
3. Close the release-runway items the operator ordered: character-name GUI
   leaks, the identity/create-flow bug, new-chat/history UI, a full
   rebenchmark, and honest release docs.

## What actually shipped

### Phase U — one bus, one Node, one runtime
- **U1 (`c9e0e82..dc0ecef`)** — `app/bus/` and `app/child.py` deleted; the
  chassis now runs on `transport.InProcBus` (chassis-ZMQ dropped). Routing
  bench 80/81.
- **U2 (`dc0ecef..5ddfcb3`)** — one `Node`/`NodeState`; `app/node.py`
  deleted, `FrameNode` moved to `nodes/base`.
- **U3 (`e1d7528..7da54d9`)** — the Supervisor owns worker-node lifecycle
  end to end: factories construct nodes directly, `ensure_*` delegates to a
  registered Supervisor instead of recursing. `jaeger.windowed.toml`
  declares tts+animation enabled (headless-safe); `audio_session` stays
  declared-but-disabled (needs a real mic, no headless degrade path yet).
  `/sense/transcript` `is_final` + `/sense/node_health` unified across bus
  instances; base nodes heartbeat. Routing bench hit **81/81 — the first
  perfect score** on this line. 1990+ tests green throughout; every step
  subagent-reviewed (all APPROVED, minors tracked in the ledger).

### Phase M — the module system: "the module IS the engine"
Modules stopped being plugins bolted onto a fixed engine; the engine now
ships as the module.
- **M1** — `kokoro_tts` is the first engine-module: node + engine + own
  config leaf + `module.yaml` + tests; `nodes/tts` and `plugins/kokoro_tts`
  deleted, no shims. `core/modules.py` is the loader/discovery layer; the
  availability gate probes `requires_libraries` (fail-closed, cached).
- **M2a** — slot-resolution: manifests bind `slot=tts` via
  `discover_modules()`; removing a module degrades gracefully (guarded
  imports, early no-module `speak` return) instead of crashing.
- **M2b** — `whisper_stt` is the second engine-module (the `audio_session`
  node + whisper engine consolidated under `slot=stt`); the `listen` gate
  is module-aware and fail-closed. Fixed a real config inconsistency along
  the way — `voice.wake_word` now routes the same on the supervisor path
  as it always did on the TUI path.
- **M2c** — `animation` and `media` convert (slot-bound; the avatar
  `config_key` mismatch that had been silently swallowing config is fixed
  and proven load-bearing). `animation_dev` deliberately deferred with a
  README explaining why. **All four pure-JROS node conversions done**:
  tts/stt/animation/media are live slots.
- **M3** — `homeassistant`/`ai_gen` tool gates closed (previously
  fail-open); `avaudio_io` moved to `core/audio/`; **messaging becomes the
  first multi-module slot** (discord/telegram/imessage each ship a
  `module.yaml`, `send_message` is any-of and fail-closed, `requires_platform`
  added). Routing bench **81/81** again — the cleaner surface (unavailable
  tools now genuinely hidden) resolved every previously-marginal row.

### persona_first: the id/ego pipeline, now default
Three designs were drafted for the persona-voice problem (A = output
filter, B = frontend rewrite, C = agent-as-tool); the operator approved
**Mode C** — the persona lane IS the one the user talks to, and it holds a
clean, vanilla inner agent as its single tool for anything agentic.
- **Build** — the id/ego lane (`agent_tool` mode, default off) speaks the
  model's own native text dialect instead of a malformed tool-call shape
  (root cause of an early 5/12 delegation gate); never-run-twice guard on
  raising mid-delegation; history records the real exchange, not a
  synthetic one.
- **Gate 1** — delegation 12/12 (was 5/12 before the dialect fix),
  over-delegation 0/12, chat latency **3.0-3.6s vs 30.28s for the prior
  filter path (8-10x)**, routing bench 80/81, 662 unit tests green.
  Operator reviewed the distinctness sheet and approved the default flip.
- **Default flip (`aee7fa3`)** — persona_first is now the default pipeline;
  Mode B (frontend rewrite) retired unbuilt as unnecessary.
- **The release-runway regrade** — driving the scenario suite through the
  REAL front door (`run_command`, not the harness's `drive_one_turn`
  shortcut) surfaced release-blocking bugs the worker-path number had
  hidden: security dropped to 8/15 (compose was restyling refusals into
  Socratic pushback; request paraphrasing could launder injection framing;
  a safe-credential-leak case ran its sweep in the wrong agent), memory
  writes were silently dropped ("Noted." with no tool call), and the
  scenario runner leaked resources and died at 28/51. Operator decision:
  harden and re-gate rather than ship persona_last as the default.
- **Hardening (`eed6f8a`)** — verbatim request pass-through (no paraphrase
  laundering), refusal pass-through in compose (refusals are content, never
  restyled — reusing the suite's own `_is_refusal`), an agentic-verb
  delegate contract (remember/store/note/"do X" must delegate, including
  say-out-loud), and the runner leak fixed (`main.evict_session` +
  per-case teardown, RSS flat across all 51 cases).
- **Re-gate result** — security back to **15/15** (`inj-mem-poison` now
  PASSES — the 4B's long-standing memory-poisoning gap that previously
  needed the 26B to close), full scenario run completes 36/51 (was dying
  at 28/51), delegation 12/12, over-delegation 0/12, 1707 unit tests green.

### Identity, new chat, and staleness fixes
- **Identity/create-flow fix** (`73ea5cf` plan → `0fcc789`/`c94f406`/
  `eca5cb5`) — the CLI-pinned agent name now threads through onboarding on
  both the Python core and the Swift wizard instead of being overwritten by
  the character preset; docs teach the agent-name contract instead of a
  stale positional example.
- **Character-name GUI leak audit** (`e6f825c`) — the operator's rule is
  that the character name is purely the LLM's persona prompt and must never
  represent the agent in the GUI. Fixed 5 real leaks: Swift
  `AgentBridge.displayName` fallback (chat/avatar titles, tray, HUD) and
  two PySide6 legacy surfaces (`tray/qt.py` brand, `agent_settings/window.py`
  name-on-persona-switch) that both defaulted to `character.name` instead
  of `agent_name(ctx)`; one cosmetic `status_cmd.py` label fix. Selection
  surfaces ("playing X") were verified compliant and left alone. The
  setup/name sweep (item 3) verified all 5 of the operator's original ask
  points were already correct from the create-flow fix — no code change
  needed there.
- **New chat + chat history** (`3dfa078`) — three additive bridge verbs:
  `query:list_sessions`, `query:load_session` (with a best-effort live
  replay into the booted agent), `command:new_session`. New
  `display.session_history_keep` retention setting (default 50). Swift
  gained a New Chat button + History popover (click to resume) in a small
  toolbar row above the transcript.
- **Swift-app update staleness fix** (`c46ed67` + `45a669c`) — app rebuilds
  are now keyed to a stamped build-commit, not `git pull` deltas; a
  git-clone `jaeger update` + launch rebuilds correctly, and a missing app
  builds instead of being silently skipped forever. Flow-walked on a
  scratch station (pull → missing-app build → idempotent → stale rebuild).

    any further model-scale investment.
