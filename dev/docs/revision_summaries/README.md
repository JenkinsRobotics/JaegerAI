# dev/docs index вЂ” what's what, by version + status

You collect docs across sessions and over time it gets hard to tell
which ones are still relevant.  This index categorises every file
under `dev/docs/` so you don't have to read each one to know
whether it still matters.

**Status legend:**

- рџџў **CURRENT** вЂ” the doc describes how JROS works today
- рџџЎ **PLAN** вЂ” a roadmap / design plan that's been implemented (some

  reviews, audits)
- рџ”ґ **WALKED BACK** вЂ” the doc describes work we DID then UNDID;

## How to use this

1. **Lost on a current behaviour?** Find it in the "Architecture +
   design" or "Subsystem-specific" rows above.
2. **Wondering why JROS made some choice?** Skip to the matching
   per-version summary (`0.X.0.md`) and look for "what we tried and
   abandoned" вЂ” many choices are reactions to a failure.
3. **Considering a change?** Check `naming_conventions.md`,
   `lean_surface.md`, and `main_loop_review.md` first.  These are
   the load-bearing current docs.
4. **Looking for unused / stale docs to delete?** Anything marked
   рџ”ґ WALKED BACK is a candidate.  Don't delete рџ”µ REFERENCE вЂ” the
   archeology has cost-zero storage but real future value.
